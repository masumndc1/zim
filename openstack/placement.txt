... how you set sync_on_startup = True in the [placement_database] section
... of /etc/placement/placement.conf
...
... To set this using puppet-placement, you don't actually need to edit the
... file manually. Because most OpenStack Puppet modules are built on a consistent
... framework, you use a specific "config" resource to inject settings into
... the .conf files.
...
... you need to have puppet-placement module in Puppetfile.

... In your Puppet manifest (e.g., site.pp or a profile module),
...  add this block:

placement_config {
  'placement_database/sync_on_startup': value => true;
}
