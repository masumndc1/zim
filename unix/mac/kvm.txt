

... how to have qemu/kvm in mac
... source: https://medium.com/@aryangodara_19887/qemu-virt-manager-and-libvirt-on-macos-with-apple-silicon-m2-dc677e6b8559
...
... first install qemu
brew install qemu
sudo port install qemu

... install libvirt
brew install libvirt

... install virt-manager, you will need macports for this
sudo port install virt-manager

... Start libvirt services (to start the libvirt daemon)
brew services start libvirt

... you need to install xquartz package for mac from
... https://www.xquartz.org/

... start virt-manager
virt-manager -c "qemu:///session" --no-fork

... error: libvirtd authentication unavailable
...
... you may need to start polkit manually by either of the following
... this might need in i3, hyperland based desktop manager.

/usr/lib/polkit-gnome/polkit-gnome-authentication-agent-1 &
/usr/libexec/polkit-mate-authentication-agent-1 &

... and your user should be added to libvirt group.

sudo usermod -aG libvirt $USER

sudo virsh net-list --all

sudo virsh net-start default
sudo virsh net-autostart default

sudo firewall-cmd --permanent --add-interface=virbr0
sudo firewall-cmd --reload

... when installing VM, try to put your iso in /tmp folder
... otherwise then VM cant be installed in system space
... which may problematic to select the interface for VM.
